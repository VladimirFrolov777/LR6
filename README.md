# Лабораторная работа №6

## Цель лабораторной работы

Изучение базовых возможностей системы управления версиями, опыт работы с Git Api, опыт работы с локальным и удалённым репозиторием.

## Ход работы

В ходе выполнения лабораторной работы №6 была изучена система контроля версий **Git** и базовые операции работы с локальным и удалённым репозиторием: 
был создан аккаунт на **GitHub**, выполнен форк репозитория преподавателя, затем установлен **Git** с официального сайта и выполнена его настройка через команды `git config --global user.name` и `git config --global user.email`,
, скриншоты сохранены в папке *img*; после настройки локальный репозиторий был создан командой `git clone`,
рабочая директория открыта, а `git pull` подтянул изменения, добавленные ранее через интерфейс GitHub, скриншоты процесса также помещены в *img*; далее была создана новая ветка командой `git checkout -b feature`,
внесены изменения в файл **README.md**, добавлены в индекс (`git add`) и зафиксированы (`git commit -m`), затем работа была продолжена в ветке **master**,
где тот же файл был изменён, что вызвало расхождение; после выполнения изменений было произведено слияние `git merge feature` в ветке master, возник конфликт («CONFLICT (content): Merge conflict in README.md»),
файл был открыт, вручную отредактирован, удалены служебные строки `<<<<<<<`, `=======`, `>>>>>>>`, затем изменения были добавлены (`git add README.md`)
и зафиксированы (`git commit -m "Разрешён конфликт при merge feature в master"`), скриншоты конфликта и его устранения размещены в *img*; после успешного слияния ветка feature удалена командой`git branch -d feature`,
затем создан файл **changes.txt**, в который поочерёдно вносились небольшие изменения с последующими `git add` и `git commit -m`, формируя цепочку коммитов, после чего выполнен откат последнего
коммита командой `git reset --hard HEAD~1`,вернувший репозиторий к предыдущему состоянию, а скриншоты действий помещены в папку *img*.
Для оформления отчёта была создана отдельная ветка report командой git checkout -b report. В этой ветке началось формирование итоговой документации, включая логи, команды и скриншоты.
История операций была получена в отформатированном виде при помощи команды: git log --pretty=format:"%h | %ad | %an | %s" --date=short Лог всех использованных команд (без вывода): git config --global user.name

фиксация изменений, просмотр логов, разрешение конфликтов, откат изменений. Также была оформлена документация в формате Markdown, что соответствует требованиям работы.
